线程概述

  • 线程是独立运行的程序，需要被触发，可以结束或者不结束。
  • 硬件模型中都是always语句块，这些线程会一直占用仿真资源，不会结束
  • 软件测试平台中的环境需要由initial语句创建，仿真过程中对象可以创建和销毁，对资源的占用是动态的

线程的使用

  • begin...end中的语句以顺序方式执行，fork...join中的语句以并发方式执行
  • 线程的执行轨迹是呈树状结构，任何的线程都有父线程
  • 父线程可以开辟若干个子线程，父线程可以暂停或者终止子线程
  • 子线程终止时，父线程可以继续执行。父线程终止时，开辟的所有子线程都应当会终止

线程的控制

  • fork并行线程语句块
  • 可以使用wait fork等待所有子线程结束
task run_threads;
    ...
    fork
        a;
        b;
        c;
    join_none
    ...
    wait fork;
endtask
  • 可以使用disable来指定需要停止的线程，如果多个线程同名，会被同时关闭掉
task check_trans
    fork: timeout_block
        begin
            ...
        end
        #TIME_OUT $display("TIMEOUT");
    join_any
    disable timeout_block;
endtask
  • disable fork可以停止从当前线程中衍生出来的所有子线程
initial begin
    check_trans() //线程0
    fork // 线程1
        begin
            check_trans(); //线程2
            fork //线程3
                check_trans();
            join
            #(TIME_OUT/2) disable fork; //停止线程1-3，单独保留线程0
        end
    join
end

线程间的通信

概述

  • 所有线程需要同步并交换数据，称之为线程间的通信（IPC）
  • 一个线程需要等待另一个
  • 多个线程可能同时访问同一个资源
  • 线程之间需要交换数据

event事件

  • 其他线程可以通过 -> 操作符来触发事件，结束对第一个线程的阻塞
event e1, e2;
initial begin
    $display("%0t: 1: before trigger", $time);
    -> e1;
    @e2; // wait (e2.triggered());
    $display("%0t: 1: after trigger", $time);
end
initial begin
    $display("%0t: 2: before trigger", $time);
    -> e2;
    @e1; // wait (e1.triggered());
    $display("%0t: 2: after trigger", $time);
end
// @0: 1: before trigger
// @0: 2: before trigger
// @0: 1: after trigger
// 第一个初始化块启动，触发e1事件，阻塞在e2上
// 第二个初始化块启动，触发e2事件，阻塞在e1上
// e1和e2在同一时刻被触发，但由于delta cycle的时间差使得两个初始化块可能无法等到e1或者e2
// 更安全的方式可以使用event的方法 triggered()
  • wait与@相比，可以避免在相同时刻触发event而带来的竞争问题，但同样无法捕捉已经被触发的事件。

semaphore

  • semaphore可以实现对同一资源的访问控制。
  • 三种基本操作：new() 方法可以创建一个带单个或者多个钥匙的semaphore， get() 可以获取一个或者多个钥匙，put() 可以返回一个或者多个钥匙。
  • 可以使用 try_get() 试图获取一个semaphore而不被阻塞，它返回1表示有足够多的钥匙，返回0表示钥匙不够。
semaphore sem;
initial begin
    sem = new(1);
    fork //多个总线事务线程，只有拿到钥匙的那一个能对总线进行驱动
        sequencer();
        sequencer();
    join
end

task sequencer;
    repeat($urandom%10) @bus.cb;
    sendTrans();
endtask

task sendTrans;
    sem.get(1);
    @bus.cb;
    bus.cb.addr <= t.addr;
    ...
    sem.put(1);
endtask
  • 线程之间的共享资源应该遵循互斥访问原则，避免出现多个线程对同一资源的访问。
  • semaphore get 到的钥匙取决于里面存了多少，put 可以无限往里放钥匙。

mailbox

  • mailbox是一种对象，需要使用 new() 来例化，例化时有一个可选的参数size来限定其存储的最大数量，如果size是0或者没有指定，信箱可以无限大。
  • put() 把数据放入mailbox，信箱为满会发生阻塞，get() 从信箱移除数据，信箱为空会发生阻塞，peek() 可以获取信箱里的数据的拷贝而不删除它。
  • try_put() ， try_get() 和 try_peek() 不会发生阻塞
  • 可以显示限定mailbox中元素的类型，可以通过 mailbox #(type = T) 来声明，例如int类型：mailbox #(int)

通信要素的比较

  • event：最小信息量的触发，单一的通知功能
  • semaphore：保护共享资源，多线程间对某一公共资源做访问可以使用这个要素
  • mailbox：精小的SV原生FIFO，线程之间做数据通信或者内部数据缓存

  1. arbiter送到slave_fifo的ack信号是由formatter送来的ack直接送走的
  2. arbiter中对data没有缓存，slave送来的data会直接送到formatter

 class fmt_driver;
    local string name;
    local virtual fmt_intf intf;
    mailbox #(fmt_trans) req_mb;
    mailbox #(fmt_trans) rsp_mb;

    local mailbox #(bit[31:0]) fifo;
    local int fifo_bound;
    local int data_consum_peroid;

  
    function new(string name = "fmt_driver");
      this.name = name;
      this.fifo = new();
      this.fifo_bound = 4096;
      this.data_consum_peroid = 1;
    endfunction
  
    function void set_interface(virtual fmt_intf intf);
      if(intf == null)
        $error("interface handle is NULL, please check if target interface has been intantiated");
      else
        this.intf = intf;
    endfunction

    task run();
      fork
        this.do_receive();
        this.do_consume();
        this.do_config();
        this.do_reset();
      join
    endtask

    task do_config();
      fmt_trans req, rsp;
      forever begin
        this.req_mb.get(req);
        case(req.fifo)
          SHORT_FIFO: this.fifo_bound = 64;
          MED_FIFO: this.fifo_bound = 256;
          LONG_FIFO: this.fifo_bound = 512;
          ULTRA_FIFO: this.fifo_bound = 2048;
        endcase
        this.fifo = new(this.fifo_bound);
        case(req.bandwidth)
          LOW_WIDTH: this.data_consum_peroid = 8;
          MED_WIDTH: this.data_consum_peroid = 4;
          HIGH_WIDTH: this.data_consum_peroid = 2;
          ULTRA_WIDTH: this.data_consum_peroid = 1;
        endcase
        rsp = req.clone();
        rsp.rsp = 1;
        this.rsp_mb.put(rsp);
      end
    endtask

    task do_reset();
      forever begin
        @(negedge intf.rstn) 
        intf.fmt_grant <= 0;
      end
    endtask

    task do_receive();
      forever begin
        @(posedge intf.fmt_req);
        forever begin
          @(posedge intf.clk);
          if((this.fifo_bound-this.fifo.num()) >= intf.fmt_length)
            break;
        end
        intf.drv_ck.fmt_grant <= 1;
        @(posedge intf.fmt_start);
        fork
          begin
            @(posedge intf.clk);
            intf.drv_ck.fmt_grant <= 0;
          end
        join_none
        repeat(intf.fmt_length) begin
          @(negedge intf.clk);
          this.fifo.put(intf.fmt_data);
        end
      end
    endtask

    task do_consume();
      bit[31:0] data;
      forever begin
        void'(this.fifo.try_get(data));
        repeat($urandom_range(1, this.data_consum_peroid)) @(posedge intf.clk);
      end
    endtask
  endclass
